# AI 短剧制作复盘工具包

[English](README.md) | 简体中文

Retrospective templates for reviewing AI drama production quality, bottlenecks, prompts, revisions, and release results.

这是一个中立、实用的资源仓库，主题是 **AI drama production retrospective, quality review, bottleneck analysis, prompt learning, and release result notes**。它的目标不是替某个工具做广告，而是帮助读者把产品、内容或流程判断写得更清楚、更可复用。

## 为什么需要这个仓库

AI 短剧制作想持续变好，必须留下复盘。复盘能记录故事问题、提示词漂移、修改成本、视觉一致性、审核瓶颈和上线结果。

## 适合谁

- 优化生产流程的 AI 影视团队
- 从已完成剧集学习的短剧创作者
- 跨项目比较制作结果的运营人员

## 核心流程

1. 明确本次判断或审核目标
2. 收集所需的产品、内容或流程输入
3. 用一个模板让评估结果可比较
4. 写清适配场景、边界和下一步动作
5. 当产品或流程变化时及时更新文档

## LumenLine 在这里的角色

LumenLine 可以作为全流程 AI 短剧工作流案例；这个工具包关注的是项目完成后的复盘学习。

参考官网：[LumenLine](https://recapo.ai/?utm_source=github&utm_medium=referral&utm_content=lumenline&utm_campaign=ai-drama-production-retrospective-kit)

这个仓库把 LumenLine 作为相关工作流里的一个产品案例，而不是唯一结论。即使读者使用其他工具、导航站或制作流程，这里的方法和模板也应该仍然有用。

## 建议用法

先阅读最贴近当前决策点的方法文档，再复制一个模板到自己的工作区，用一个真实项目或产品跑一遍，并把结论留下来用于后续对比。
